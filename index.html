Feature 1: Mutual Friends

Algorithm: Set Intersection
Time Complexity
O(min(deg1, deg2))
Space Complexity
O(min(deg1, deg2))
  • Get friend sets for both users
  • Find intersection (common friends)
  • Calculate Jaccard similarity: |intersection| / |union|
  • Return mutual friends and similarity ratio

Feature 2: Influencer Leaderboard

Algorithm: PageRank + Weighted Scoring
Time Complexity
O(V √ó log(K) + E √ó I)
Space Complexity
O(V + K)
  • Calculate PageRank scores (importance based on followers)
  • Compute influence from: Followers (30%), Fans (35%), Friends (15%), PageRank (25%)
  • Use min-heap to maintain top K influencers efficiently

Feature 3: Community Detection

Algorithm: Label Propagation Algorithm (LPA)
Time Complexity
O(V √ó avg_degree √ó iter)
Space Complexity
O(V)
  • Initialize: Each node gets its own ID as label
  • Iterate: For each node, adopt most frequent friend's label
  • Repeat until convergence or max iterations (10)
  • Group nodes with same label into communities
  • Calculate internal edge density for each community

Community Detection Visualization

Label Propagation spreads community labels through the network iteratively until stable communities emerge

Feature 4: Shortest Path

Algorithm: Bidirectional BFS
Time Complexity
O(V + E)
Space Complexity
O(V)
  • Check if nodes are directly connected
  • Use Bidirectional BFS: search from both ends
  • Meet in the middle ‚Üí 50% faster than standard BFS
  • Fallback to unidirectional BFS if needed
  • Cache results for repeated queries

Feature 5: Friend Recommendations

Algorithm: Friends-of-Friends + Multi-Factor Scoring
Time Complexity
O(V √ó avg_degree¬≤)
Space Complexity
O(V)
  • Find candidates: friends of friends (excluding existing friends)
  • Score using: Mutual friends (40%), Common interests (25%)
  • Geographic proximity (20%), Community similarity (15%)
  • Sort by total score and return top K recommendations

Feature 6: Friendship Score

Algorithm: Multi-Factor Weighted Scoring
Time Complexity
O(avg_degree + M)
Space Complexity
O(avg_degree)
  • Check if users are friends (mutual connection check)
  • Base score: 1.5 for friends, 2.5 for non-friends
  • Apply weighted factors: Mutual friends (30%), Messages (25%)
  • Transitive scores (15%), Connection time (15%), Location & Interests
  • Normalize to [1.0, 3.0] range - lower means stronger friendship

Feature 7: User Search (Autocomplete)

Algorithm: Trie Data Structure
Time Complexity
Build: O(V√óL), Search: O(P+R)
Space Complexity
O(V √ó L)
  • Build phase: Insert all user names into Trie
  • Each character creates a node in the tree
  • Search phase: Navigate Trie following prefix characters
  • Collect all user IDs from matching prefix node
  • Sort results alphabetically and return top K matches
  • Case-insensitive matching for better UX

Trie Structure Advantage

Enables ultra-fast prefix matching - perfect for real-time autocomplete in search boxes

Example: Searching "Go"
Root ‚Üí G ‚Üí o
Finds: Gokul, Gordon, Govind
O(P) Time
P = prefix length = 2

2. Node Generation

3. Edge Generation
  • ‚Ä¢ Base: 2% + 15% (same region) + 10% (per shared interest)
  • ‚Ä¢ Friend (mutual) vs Fan (one-way) relationships

  • ‚Ä¢ Formula: base - (mutual_friends √ó 0.5) - (messages √ó 0.3)

  • ‚Ä¢ Daily messages: 30% of edges, 0-10 messages/day
  • ‚Ä¢ Relationship changes: friend‚Üîfan transitions
  • ‚Ä¢ New connections: 0.5% probability/day

  • ‚Ä¢ 10% nodes designated as viral
